Context file — bộ nhớ
dài hạn của dự án

File AI tự đọc ở đầu mỗi session. Giúp AI hiểu context dự án mà không cần giải thích lại mỗi lần.

Không có file

Giải thích lại tech stack, convention, quyết định thiết kế.

Có file tốt
Bắt tay vào việc ngay

Duy trì
Living document

Đọc requirement cùng AI

Đừng đọc một mình. Nhờ AI tóm tắt, liệt kê câu hỏi clarify, nhận diện dependency & risk trước khi code.

✕ Đọc một mình
  • Bỏ sót edge case ẩn trong mô tả
  • Không biết hỏi gì để clarify
  • Estimate thiếu buffer cho risk
  • Mất 2–3 giờ để hiểu hết
    • ✓ Đọc cùng AI
  • Highlight ambiguity ngay lập tức
  • Có sẵn list câu hỏi cho PM
  • Risk được nhận diện trước khi code
  • Chỉ mất 20–30 phút
    • ⊕ Rule of thumb khi ET

    AI cho baseline, bạn chốt. Thêm buffer cho phần AI chưa nắm - technical debt, performance, tích hợp bên thứ ba... Dựa vào baseline của AI để estimate, và điều chỉnh theo thực tế dự án.

    File/feature vừa sửa gần đây
    05 · Environment
    Platform, RN version, device/sim, build variant

    Self-test trước khi
    chuyển tester

    Nhờ AI sinh checklist phủ đủ 4 nhóm — để không bỏ sót test case quan trọng.

    Happy path
    Luồng chính hoạt động đúng
    Edge cases
    Giá trị biên, dữ liệu bất thường
    Negative cases
    Input sai, xử lý lỗi đúng cách
    Regression
    Không break feature liên quan
    ⊕ Phép tính đơn giản

    Bug catch ở self-test = 5 phút. Bug do tester tìm ra = thêm 30 phút (re-open → fix → re-test → close).

    Dùng context hiệu quả

    Agentic tool tự đọc file — không cần dán code như chat web. Quản context tốt tương đương với code nhanh hơn, ít lỗi hơn.

    ✕ Tốn & nhiễu
  • Dán cả file > 500 dòng
  • Hỏi nhiều câu nhỏ lẻ tẻ
  • Một session cho mọi task
  • Giải thích lại context đã có
    • ✓ Hiệu quả
  • Để AI tự đọc file (chỉ trỏ tên / @mention)
  • Gộp câu hỏi vào 1 prompt
  • Session dài → nén lại (/compact); xong hẳn → reset (/clear) hoặc tạo session mới
  • Tận dụng CLAUDE.md + /docs

    Skills — SOP mà AI
    là người thực thi

    Viết một lần (SKILL.md), AI đọc & làm theo mỗi khi gọi. Gọi /et hay /debug là chạy đúng quy trình team đã định.

    ✕ Skill kém
  • Mô tả chung chung: "hãy debug lỗi"
  • Output mơ hồ
  • Không khai báo platform
  • 1 skill làm quá nhiều việc
    • ✓ Skill tốt
  • Quy trình cụ thể từng bước
  • Output có cấu trúc (format cố định)
  • Rõ React Native, Expo, pattern
  • 1 skill = 1 mục đích rõ ràng
    • Lưu tại .claude/skills/<tên>/SKILL.md & commit — cả team dùng chung Skill chưa thử kỹ → chưa phải chuẩn
